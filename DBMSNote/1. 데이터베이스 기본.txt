1. 파일 시스템 사용
    1) 데이터 중복성 문제 : 공간 낭비
    2) 업데이트 및 데이터 일관성(data consistency) 유지에 어려움 
    3) 데이터 종속성 
        - 응용프로그램이 파일 데이터 구조에 종속적
        - 파일구조가 바뀔 때마다 응용프로그램 교체 필요 
    4) 동시성(Concurrency) 제공이 어려움         
        - 여러 사용자가 동시에 접근할 때 문제 해결에 어려움 
    5) 보안(Security) 제공 이슈        
        - 사용자 별 파일 안의 일부 데이터 읽기 권한 제어가 어려움 

2. 데이터베이스 
    1) 특정 조직의 여러 사용자가 공유하여 사용할 수 있도록 통합해서 저장한
       운영 데이터 집합    
    2) 예
        - 은행 : 계좌정보, 입출금 내역 등
        - 항공사 : 예약정보, 비행기 스케쥴 
        - 대학교 : 학생정보, 수강신청 
        - 온라인 쇼핑몰 : 고객기록, 주문내역 
        - 제조업 : 제품목록, 주문, 재고, 공급망
        - 회사 인사시스템 : 직원정보, 연봉             

3. 데이터베이스 특징                          (쇼핑몰 예)
    1) 실시간 접근 가능                       목록 조회   
    2) 계속적으로 변화                        구입정보, 물건재고 정보 
    3) 동시 공유가 가능                       많은 고객이 동시 접속, 구매 가능 
    4) 저장된 주소가 아닌                     (QUERY) 가장 많이 팔린 제품은?  
       내용으로 참조 가능   

4. 데이터베이스 유형        
    1) 정형 데이터 (structured data)
        - 엑셀의 스프레드시트, 관계형 데이터베이스의 테이블 

    2) 반정형 데이터 (semi-structured data)        
        - HTML, XML, JSON

    3) 비정형 데이터 (unstructured data)        
        - 정해진 구조가 없이 저장된 데이터 
        - text, 멀티미디어 데이터 

5. 데이터베이스 관리시스템 (DBMS)       
    1) 응용프로그램과 데이터 파일 분리
        - 프로그램은 DBMS를 통해서 파일시스템의 파일에 저장된 데이터에 접근
    2) DBMS
        - 응용프로그램과 데이터 연결을 도와주며 
          데이터 관리와 데이터에 대한 기본처리를 담당하는 
          추가적인 시스템 소프트웨어
        - 파일시스템의 데이터중복과 데이터 종속 문제를 해결하기 위해
          제시된 소프트웨어
        - 데이터베이스 생성과 관리를 담당 
        - DBMS 통해 데이터 삽입, 수정, 삭제, 업데이트, 검색        
    3) 대표적 DBMS  
        - Oracle, MS SQL Server, Postgresql, MySql..

    4) DBMS 주요기능
        - 정의 기능 : 데이터베이스 구조를 정의하거나 수행 
        - 조작 기능 : 데이터를 삽입,검색,수정,삭제(CRUD) 하는 연산을 수행
        - 제어 기능 : 데이터를 항상 정확하고 안전하게 유지하는 기능   

    5) DBMS 장점
        - 파일시스템의 데이터 중복 문제 해결
        - 데이터 독립성 확보 (데이터 구조 바뀌어도 응용프로그램을 바꿀 필요 없음)
        - 데이터 동시 공유 
        - 데이터 보안 향상 - 사용자별 접근 가능한 데이터베이스 영역 제한 기능 
        - 데이터 무결성 유지 
        - 표준화 방식으로 데이터에 접근 (SQL)     
        - 장애 발생 후 회복 시 데이터 일관성과 무결성 유지 
        - 응용 프로그램 개발 비용이 줄어 듬

    6) DBMS 단점
        - DBMS 구매 비용 
        - 응용프로그램이 DBMS를 통해 데이터에 접근하기 때문에 추가적인 
          오버헤드가 있음
        - 응용프로그래머가 DBMS가 어떻게 동작하는지와 표준 데이터 언어에 대한
          지식 필요 
        - DBMS 장애가 발생할 때 모든 응용프로그램 장애가 발생함                           

6. 데이터베이스의 중요 개념 
    1) 스키마(schema)
        - 데이터베이스에 저장되는 데이터의 논리적구조와 제약조건을 정의한 것 
        - 고객 schema
            고객번호      이름          나이     주소     
              INT      VARCHAR(10)     INT     VARCHAR(100)

    2) 인스턴스 (instance)              
        - 정의된 스키마에 따라 데이터베이스에 실제로 저장된 값

7. 3단계 데이터베이스 구조  
    1) Physical level - Internal Schema (내부 스키마)
        - 실제 데이터가 데이터베이스에 어떻게 저장되는지 기술 
    2) Conceptual level - Conceptual Schema (개념 스키마)
        - 일반적인 스키마 
    3) External level - External Schema (외부 스키마)          
        - 사용자가 보는 View level 
        - 사용자에게 view의 형태로 필요한 정보만 보여주는 단계 
        - 데이터베이스 하나에 여러 개의 외부 스키마 가능 

8. 데이터베이스 사용자 
    1) 데이터베이스 관리자 (DBA)
        - 데이터베이스 시스템을 운영하고 관리
        - 데이터베이스를 설계 및 구축, 제어 
        - DBMS 자체는 물론 데이터베이스 구축, 관리에 해박한 지식과
          많은 경험이 요구됨 
    2) 응용 프로그래머 
        - 데이터베이스 언어를 이용하여 응용프로그램을 작성   
    3) 최종 사용자 
        - 데이터베이스에 접근하여 데이터를 조작

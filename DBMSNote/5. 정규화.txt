1. 정규화(Normalization)의 개념 
    1) 데이터의 일관성을 유지하고 데이터의 중복을 방지하여 
       데이터의 유연성을 유지하기 위해 
       데이터를 분해하는 과정이라고 할수 있음.
    2) 관계형 데이터베이스의 설계에서 중복을 최소화하게 데이터를 구조화하는 프로세스.
    3) 함수적 종속성(FD: Functional Dependency)등과 같은 이론에 근거하여
       관계형 데이터베이스 테이블의 삽입/삭제/업데이트 이상(anomaly)현상 발생을
       최소화하기 위해 좀 더 작은 단위의 테이블로 설계하는 과정임.
       ----------------------------------------------------    
       즉 정규화는 데이터 모델을 정규형에 맞도록 고치는 과정임.

2. 용어
    1) 정규형 (NF: Normal Form)
        - 정규화한 결과

    2) 함수적 종속성(FD: Functional Dependency)
        - 테이블의 특정 컬럼 A의 값을 알면 다른 컬럼 B값을 알 수 있을 때, 
          컬럼 B는 컬럼 A에 함수적 종속성이 있다고 함.
        - 예
            - 주민등록번호를 알면 고객명을 알수 있을 때,
              "고객명은 주민등록번호에 함수적 종속성이 있다"

    3) 결정자 (Deteminant)
        - 함수적 종속성에서 컬럼 A를 결정자라고 함.
        - 예
            - 주민번호는 고객명을 결정하므로 주민번호는 고객명의 결정자임. 

3. 정규화 효과 
    1) 상호 종속성이 강한 데이터 요소들을 분리, 독립된 개념(엔터티, 테이블)으로
       정의하게 됨에 따라 높은 응집도 & 낮은 결합도 
       (High Cohesion & Loose Coupling) 원칙에 충실해지며
       이로 인해 유연성이 극대화 됨.              
    2) 정규화를 통해 재활용 가능성이 높아짐      
    3) 정규화는 식별자가 아닌 속성이 한 번만 표현됨에 따라 
       중복이 최소화됨. 성능이 향상됨.              

4. 정류화 이론 
    1) 1차 정규화 => 제1정규형
        - 한 속성에 여러 개의 속성값을 갖거나 같은 유형의 속성이 여러 개인 경우 
          해당 속성을 분리시킴
        - 1차 정규화 작업으로 속성의 원자성을 확보하게 됨   
    2) 2차 정규화 => 제2정규형 
        - 주식별자에 완전 함수 종속되지 않은 속성을 분리함 
        - 제2정규형은 제1정규형을 만족하고 엔터티내에서 식별자가 아닌 속성은 
          식별자 전체에 종속되어야 함
        - 식별자 전체에 종속적이지 않거나 
          복합식별자일 경우 식별자중 일부 식별자 속성(들)에만 종속적인 일반 속성은 
          분리되어야 함   
        - 2차 정규화 작업으로 부분 종속 속성이 된 속성을 분리함    

    3) 3차 정규화 => 제3정규형
        - 일반 속성끼리 함수 종속이 발행한 속성을 분리함 
        - 제3정규형은 제2정규형을 만족하고 일반 속성들 간에도 함수 종속 관계가
          존재하지 않아야 함 
        - 제3정규형을 만족하려면 일반 속성들 간 종속관계가 존재하는 것들은
          분리되어야 함    

5. 함수적 종속성 (Functional Dependency)에 근거한 정규화 수행
    1) 함수적 종속성은 데이터들이 어떤 기준값에 의해 종속되는 현상을 지칭하는 것.
    2) 이때 기준값을 결정자(Deteminant)라 하고 종속되는 값을 종속자(Dependent)라함.
    3) 예
        - 주민등록번호 ================> 이름,출생지,주소  
          (결정자)                         (종속자)

        - 이름,출생지,주소는 주민등록번호에 함수적 종속성을 가지고 있음.
        - "주민등록번호가 이름,출생지,주소를 함수적으로 결정한다"
    4) 정규화 작업은 이러한 함수적 종속성을 만족시키는 작업임.

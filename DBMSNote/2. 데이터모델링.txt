1. 데이터 모델링 (Data Modeling)
    1) 테이블(데이터)의 근간이 되는 것 = 데이터 모델 
    2) 데이터 모델을 설계하는 과정 = 데이터 모델링 

2. 모델링 
    1) 우리가 살고 있는 3차원의 현실 세계를 단순화하여 표현하는 것 
    2) 현실 세계를 추상화하여 그 구조를 표현한 것    
    3) 현실 세계와 연관된 주제를 위하여 명확하게 하는 것  
    
    => 모델링이란 복잡한 현실 세계를 추상화,단순화,명확화하기 위해 
       일정한 표기법으로 모델을 표현하는 기법 

3. 데이터 모델링 
    1) 모델링 앞에 데이터라는 단어가 붙으면 데이터 모델링이 됨 
    2) 현실 세계의 비즈니스를 IT 시스템으로 구현하기 위해 
       데이터 관점으로 업무를 분석하는 기법 
    3) 현실 세계의 비즈니스를 약속된 표기법으로 표현하는 과정 
    4) IT 시스템의 근간이 되는 데이터베이스를 구축하기 위한 분석 및 
       설계의 과정 
    5) 대부분의 IT 시스템은 관계형 데이터베이스(RDBMS) 기반으로 구축되어 있음
    6) 데이터 모델링을 기반으로 물리적인 데이터베이스가 구축되고 
       SQL문을 활용하여 데이터가 입력(INSERT),수정(UPDATE),삭제(DELETE),
       조회(SELECT)되고 있음.                 

4. 데이터 모델링의 3단계 
                  개념 데이터 모델링 (추상적)
    현실세계    ===============================>   개념적 구조 
                                                        ||
                                                        || 논리 데이터 모델링 
                  물리 데이터 모델링 (구체적)            \||/
    물리구조   <================================   논리적 구조     
    (데이터베이스)

    1) 개념적 데이터 모델링 
        - 포괄적인 수준의 데이터 모델링 진행
    2) 논리적 데이터 모델링   
        - IT 시스템에서 구현하고자 하는 비즈니스를 만족하기 위해 
          기본키, 속성, 외래키 등을 정확하게 표현하는 단계  
    3) 물리적 데이터 모델링 
        - 논리 데이터 모델을 기반으로 실제 물리 DB 구축을 위해
          설계하는 단계     

5. 데이터 모델링의 3요소 
    1) 어떤 것(Thing)
                        복수/집합 개념              단수/개별 개념 
            ------------------------------------------------------------                        
        -            엔터티(Entity)                 인스턴스(Instance)
                     예) 사원                        예) 이순신 사원 

    2) 어떤 것의 성격 (Characteristic of a Thing)

        -            속성(Attribute)                    속성값(Attribute value)    
                     예) 사원은 사원번호,사원명을 가진다   예) 사원번호 1번은                   
                                                            이순신이다 

    3) 어떤 것 간의 연관 (Association between Thing)

        -            관계(Relationship)               패어링(Pairing)   
                     예) 사원은 부서에 속한다             예) 이순신 사원은     
                                                            개발팀에 속한다                                                   

6. ERD (Entity Relationship Diagram)를 그리는 순서 

    1) 비즈니스에 필요한 엔터티를 그리고 속성을 정의함 
    2) 1번에서 그린 엔터티를 적절하게 배치함 
    3) 각 엔터티 간의 관계를 설정함 
    4) 설정한 관계의 관계명을 기술
    5) 설정한 관계의 참여도를 기술 
    6) 설정한 관계의 필수 여부를 기술

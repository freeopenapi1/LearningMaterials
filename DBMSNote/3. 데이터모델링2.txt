1. 엔터티(Entity)

    1) 개념
        - 사람, 사물, 사건, 개념 등의 명사에 해당함
        - 비즈니스 관점에서 IT 시스템을 통해 관리가 필요한 관심사에 해당함 
        - 결국 비즈니스를 구현하기 위해 저장해야 하는 어떤 것(Thing).

    2) 엔터티와 인스턴스 
        - 하나의 엔터티는 여러 개의 인스턴스를 가짐.
        - 엔터티는 인스턴스의 집합이라고 할 수 있음.
        - 예) 
                    엔터티              인스턴스
                  -----------------------------------------
                     상가              스타벅스 천호로대호점   
                                       스타벅스 강남점 
                    지하철역            8호선 천호역
                                       2호선 잠실역                                        
    
    3) 엔터티 표기법 
        - 모서리가 둥근 사각형(바커 표기법 기준)
        - 맨 위에 엔터티명 기재함 

    4) 엔터티 특징 
        - 유일한 식별자에 의해 식별이 가능해야 함
            - 집합 내에서 단 1건을 콕 짚어낼 수 있어야 함 (상가번호, 지하철역번호)        
        - 인스턴스(2개 이상)의 집합이여야 함 
        - 비즈니스 프로세스에 의해 반드시 이용되어야 함 
        - 반드시 속성을 가지고 있어야 함 
            - 속성이 존재하지 않는 것은 엔터티가 될수 없음
        - 다른 엔터티와 최소 1개 이상의 관계가 있어야 함  

    5) 엔터티 분류1 (유무형)
        - 유형
            - 실체가 존재하고 물리적인 행태가 있음
            - 안정적이고 지속적으로 활용되는 엔터티.
            - 예) 고객    
        - 개념
            - 비즈니스적으로 관리해야 할 개념적 정보를 저장하는 엔터티 
            - 예) 보험상품                     
        - 사건 
            - 비즈니스 수행함으로써 발생되는 엔터티 
            - 유형/개념 엔터티에 비해 데이터 발생량이 많음
            - 예) 계약 

    6) 엔터티 분류2 (발생시점)        
        - 기본 (Key, 키)
            - 스스로 태어난 존재에 대한 정보
            - 타 엔터티와의 관계에 의해서 생성되는 것이 아닌 독립적으로 생성
              가능한 엔터티 
            - 중심엔터티의 부모 역할을 하게 됨  
            - 예) 고객, 상품   
        - 중심 (Main, 메인)
            - 기본 엔터티로부터 발생됨
            - 비즈니스에 있어서 중심적인 역할을 하는 엔터티 
            - 예) 주문  
        - 행위 (Action, 액션)
            - 2개 이상의 부모 엔터티로부터 발생되는 엔터티 
            - 상세 설계 단계 혹은 프로세스와 상관 있는 모델링을 진행하면서 도출됨                        
            - 예) 주문목록

    7) 엔터티 명명
        - 가급적 업무 담당자들(현업)이 사용하는 용어를 사용
        - 엔터티는 단수명사이여야 함 
        - 엔터티의 이름은 해당 모델 내에서 유일한 이름이어야 함 
        - 엔터티의 생성 의미에 맞게 이름을 부여함             

2. 속성 (Attribute)       
    1) 비즈니스에서 필요로 함
    2) 인스턴스에서 관리하고자 하는 의미임 
    3) 더 이상 분리되지 않는 최소의 데이터 단위

    4) 엔터티, 인스턴스, 속성, 속성값의 관계
        - 1개의 엔터티는 여러 개의 인스턴스를 가질 수 있음
            - 하나의 인스턴스는 여러 개의 속성을 가짐 
            - 하나의 속성은 단 하나의 속성값을 가지게 됨 
        - 1개의 엔터티는 2개 이상의 속성을 갖음 
        - 예) 지하철역(엔터티)은 여러 개의 역이 있다.(인스턴스)
              지하철역에 대한 정보는 선명, 역명이 있다.(속성)
              선명 속성의 속성값은 '8호선'이고,
              역명 속성의 속성값은 '천호'역이 있다 (속성값)

    5) 속성 표기법 
        - "#"를 붙여 식별자 임을 표시  
        - "*"를 붙여 필수값 임을 표시 
        - "∘"를 붙여 선택값 임을 표시            

    6) 속성의 특징
        - 엔터티와 마찬가지로 반드시 비즈니스에서 필요로하고 
          IT 시스템에서 저장 및 관리하고자 하는 정보여야 함.
        - 정규화 이론에 따라 속성이 속해 있는 엔터티의 주식별자에 
          함수적 종속성을 가져야 함

            - 예) 지하철역 (엔터티)
                  ---------------------
                  지하철역번호(주식별자)     => 지역철역번호가 선명과 역명을 결정함   
                  선명
                  역명
        - 하나의 속성에는 한 개의 값만을 가짐  

    7) 속성의 분류1 (특성)                         
        - 기본속성 (Basic Attribute)
            - 비즈니스 분석을 통해 도출된 속성
            - 예) 상가 엔터티의 상호명 속성은 기본 속성임 
        - 설계속성 (Designed Attribute)
            - 데이터 모델 설계를 하면서 도출하는 속성
            - 예) 상가 엔터티의 표준산업분류코드 속성은 설계속성이라고 할수 있음.
        - 파생속성 (Derived Attribute)
            - 다른 속성에 의해서 계산이나 변형이 되어 생성되는 속성
            - 예) 상가 엔터티의 주소정보를 기반으로 위도, 경도 속성을 파생속성이라함                  

    8) 속성의 분류2 (엔터티 구성방식)
        - PK (Primary Key) 속성
            - 엔터티에서 단 하나의 인스턴스를 식별할 수 있는 속성
            - 예) 상가 엔터티의 상가번호 속성은 PK 속성이라고 할수 있음 
        - FK (Foreign Key, 외래키) 속성 
            - 타엔터티와의 관계를 통해 포함된 속성
            - 예) 지하철역승하차 엔터티의 지하철역명 속성은 FK 속성이라고 할수 있음.                       
        - 일반 속성 
            - 엔터티내에 존재하면서 PK 혹은 FK 속성이 아닌 속성 
            - 예) 상가 엔터티의 상호명 속성은 일반 속성이라고 할수 있음.  

    9) 속성의 분류2 예

            엔터티명            속성분류            속성명
       ===========================================================
            지하철역             PK 속성          지하철역번호
                                일반 속성         선명
                                                  역명
       ----------------------------------------------------------- 
            지하철역승하차       PK 속성          지하철역번호
                                                 기준일자
                                                 시작시간
                                                 종료시간
                                                 승하차구분코드   
                                FK 속성          지하철역번호  
                                일반 속성         승하차횟수      
                             
    10) 도메인 (Domain)
        - 속성이 가질 수 있는 값의 범위를 도메인이라 부름 
        - 예
            - 학생 엔터티의 학점 속성의 도메인 
                - 0.0 ~ 4.5 범위를 갖는 실수값
            - 학생 엔터티의 핸드폰번호 속성의 도메인 
                - 길이가 20자리 이내인 문자열 
        - 각 속성의 속성값은 정의된 도메인 이외의 값을 가질수 없음 (도메인 정의시)             

    11) 속성의 명명 
        - 비즈니스에 사용하는 이름을 부여함 
        - 속성명을 서술식으로 명명하지 않음 
        - 전체데이터 모델 내에서 유일한 이름의 속성명으로 명명하는 것이 좋음
